# 职聊凭证 AI 比赛实现计划

## 一句话定位

职聊凭证 AI 是一个面向求职者的招聘沟通承诺提取与确认系统。它把招聘聊天记录整理成可核验、可确认、可追溯的沟通确认单。

## 目标用户

1. 正在投递岗位和约面试的大学生
2. 对薪资、试用期、社保、加班、试岗存在疑问的求职者
3. 校招就业服务老师、人力服务机构、招聘平台

## MVP 范围

1. 文本粘贴分析
2. 聊天截图 OCR 入口
3. 招聘承诺结构化抽取
4. 未确认事项识别
5. 风险提醒
6. HR 追问话术
7. 沟通确认单复制和打印

## Prompt 设计

### System Prompt

```text
你是“职聊凭证 AI”的招聘沟通承诺抽取引擎。你的任务是从招聘方与求职者的聊天记录中，识别招聘沟通中的明确承诺、模糊承诺、待确认事项、潜在风险，并生成可用于双方确认的结构化 JSON。

你必须遵守以下规则：

1. 只基于用户提供的聊天记录进行判断，不得编造聊天中没有出现的信息。
2. 输出必须是严格 JSON，不得包含 Markdown、解释文字、注释或多余字段。
3. 如果某项信息没有出现，字段值使用 null。
4. 如果某项信息表达模糊，必须标记为 "uncertain"。
5. 如果某项信息已经明确表达，标记为 "confirmed"。
6. 如果聊天中出现前后矛盾、附加条件、口头承诺、含糊词语、未写入合同风险，必须写入 risks。
7. 每条承诺都要尽量保留原文证据 evidence。
8. 风险提醒要中性、克制、可核验。
9. 追问话术要礼貌、简短、适合求职者直接发送给招聘方。
10. JSON 必须符合以下结构：

{
  "conversation_meta": {
    "candidate_name": null,
    "company_name": null,
    "position_name": null,
    "recruiter_name": null,
    "chat_date": null
  },
  "commitments": [
    {
      "category": "salary | location | probation | social_insurance | working_time | job_responsibility | interview | subsidy | reimbursement | onboarding_condition | other",
      "title": "",
      "status": "confirmed | uncertain | conflicting | missing",
      "value": "",
      "conditions": [],
      "evidence": [],
      "confidence": 0.0
    }
  ],
  "confirmation_sheet": {
    "summary": "",
    "items_to_confirm": [
      {
        "item": "",
        "suggested_confirmation_text": ""
      }
    ]
  },
  "unconfirmed_items": [
    {
      "item": "",
      "reason": "",
      "follow_up_question": ""
    }
  ],
  "risks": [
    {
      "level": "low | medium | high",
      "risk_point": "",
      "reason": "",
      "suggested_action": ""
    }
  ],
  "follow_up_scripts": [
    {
      "scenario": "",
      "message": ""
    }
  ]
}
```

### User Prompt 模板

```text
请从以下招聘聊天记录中抽取招聘沟通承诺，并输出严格 JSON。

抽取范围包括：
薪资、地点、试用期、社保、公积金、工作时间、岗位职责、面试安排、补贴、报销、入职条件、绩效奖金、提成、加班、调岗、培训、背调、合同签署、其他影响求职决策的信息。

请特别关注：
1. 是否存在口头承诺。
2. 是否存在“到时候再说”“一般都有”“差不多”“看情况”“领导定”等模糊表达。
3. 是否存在前后不一致。
4. 是否存在未确认的薪资结构、试用期折扣、社保缴纳时间、工作地点变化、加班规则、入职条件。
5. 输出必须是严格 JSON。

聊天记录如下：
{{CHAT_RECORD}}
```

## 样例一

### 聊天记录

```text
招聘方：你好，我们这边是星河互动，看到你投递了用户运营岗位。
求职者：你好，想了解一下岗位待遇和工作地点。
招聘方：岗位在杭州滨江，薪资 12K 到 15K，具体看面试表现。
求职者：这个薪资是税前吗？试用期打折吗？
招聘方：税前，试用期 3 个月，试用期 80%。
求职者：社保公积金怎么交？
招聘方：五险一金都有，入职当月交，公积金按 8%。
求职者：工作时间呢？
招聘方：早 9 晚 6，双休，不过活动上线前可能要加班。
求职者：加班有调休吗？
招聘方：一般可以调休，具体看部门安排。
求职者：岗位主要做什么？
招聘方：主要负责用户活动策划、社群运营、数据复盘，还有跨部门推进活动上线。
求职者：面试流程是什么？
招聘方：一共两轮，第一轮业务主管，第二轮 HR，最快下周三可以安排线上面试。
求职者：有餐补或者交通补贴吗？
招聘方：餐补每天 25 元，交通补贴没有。
```

### 期望摘要

```json
{

  "confirmed_points": [
    "工作地点为杭州滨江",
    "薪资范围为税前 12K 到 15K",

    "面试共两轮，第一轮业务主管，第二轮 HR"

}
```

## 样例二

### 聊天记录

```text
招聘方：我们是青川智能制造，目前招招聘专员，主要负责一线工人招聘。
求职者：这个岗位在哪个城市？
招聘方：目前在苏州园区，后面可能会去昆山工厂支持一段时间。
求职者：去昆山是长期吗？
招聘方：应该不会太久，看项目招聘情况。
求职者：薪资大概多少？
招聘方：底薪 8K，加招聘绩效，做得好一个月能到 12K 左右。
求职者：绩效怎么算？
招聘方：按入职人数和留存算，具体公式入职后主管会讲。
求职者：试用期多久？
招聘方：试用期 6 个月，表现好可以提前转正。
求职者：试用期工资一样吗？
招聘方：这个岗位试用期也是 8K。
求职者：社保公积金呢？
招聘方：社保入职次月交，公积金转正后交。
求职者：工作时间是双休吗？
招聘方：大小周，早 8 点半到晚 5 点半，招聘旺季周末可能要去招聘会。
求职者：周末招聘会有补贴吗？
招聘方：市内交通可以报销，餐补看当天安排。
求职者：入职前需要什么条件？
招聘方：需要背调和体检，背调没问题就发正式 offer。
```

## 实现路线

第一阶段：

1. 完成静态 Web 原型
2. 内置样例聊天记录
3. 使用本地规则模拟 AI 抽取
4. 生成确认单和追问话术

第二阶段：

1. 增加后端接口
2. 接入 OCR
3. 接入大模型结构化抽取
4. 保存历史确认单

第三阶段：

1. PDF 导出
2. 证据时间线
3. 对话截图与文字证据绑定
4. 招聘平台或校园就业系统接入

## 答辩话术

1. 职聊凭证 AI 帮助求职者把零散聊天记录转化为清晰的沟通确认单，降低口头承诺遗漏风险。
2. 系统可自动识别薪资、试用期、社保、工作地点、加班、补贴等关键条款，让招聘沟通更透明。
3. 产品会标出未确认事项和模糊表达，并生成可直接发送给招聘方的追问话术。
4. 对招聘平台和人力服务机构而言，它能沉淀沟通证据，提升岗位信息真实性和服务可信度。
5. 这个系统面向真实求职场景，核心价值是让每一次招聘沟通都可核验、可确认、可追溯。
